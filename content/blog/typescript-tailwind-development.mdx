# TypeScriptとTailwind CSSでモダンなUI開発

モダンなWeb開発において、TypeScriptとTailwind CSSの組み合わせは非常に強力です。この記事では、両者を効果的に活用する方法について解説します。

## TypeScriptの利点

TypeScriptを使用することで、以下のメリットを得られます：

### 1. 型安全性
```typescript
interface BlogPost {
  title: string;
  date: string;
  excerpt: string;
  categories: string[];
  tags: string[];
}

function BlogCard({ post }: { post: BlogPost }) {
  // postのプロパティが型チェックされる
  return (
    <div className="bg-white rounded-lg shadow-md p-6">
      <h2 className="text-xl font-bold">{post.title}</h2>
      <p className="text-gray-600">{post.excerpt}</p>
    </div>
  );
}
```

### 2. 優れた開発体験
- インテリセンス（自動補完）
- リファクタリング支援
- エラーの早期発見

## Tailwind CSSの活用

Tailwind CSSは、ユーティリティファーストのCSSフレームワークです：

### レスポンシブデザイン
```tsx
<div className="
  grid 
  grid-cols-1 
  md:grid-cols-2 
  lg:grid-cols-3 
  gap-4
">
  {/* コンテンツ */}
</div>
```

### ダークモード対応
```tsx
<div className="
  bg-white 
  dark:bg-gray-900 
  text-gray-900 
  dark:text-white
">
  {/* コンテンツ */}
</div>
```

## 効率的な開発手法

### 1. コンポーネントの型定義
```typescript
interface ButtonProps {
  variant: 'primary' | 'secondary' | 'outline';
  size: 'sm' | 'md' | 'lg';
  children: React.ReactNode;
  onClick?: () => void;
}

const Button: React.FC<ButtonProps> = ({ 
  variant, 
  size, 
  children, 
  onClick 
}) => {
  const baseClasses = 'font-semibold rounded transition-colors';
  const variantClasses = {
    primary: 'bg-blue-600 text-white hover:bg-blue-700',
    secondary: 'bg-gray-600 text-white hover:bg-gray-700',
    outline: 'border-2 border-blue-600 text-blue-600 hover:bg-blue-50'
  };
  const sizeClasses = {
    sm: 'px-3 py-1 text-sm',
    md: 'px-4 py-2',
    lg: 'px-6 py-3 text-lg'
  };

  return (
    <button
      className={`${baseClasses} ${variantClasses[variant]} ${sizeClasses[size]}`}
      onClick={onClick}
    >
      {children}
    </button>
  );
};
```

### 2. カスタムフックの作成
```typescript
import { useState, useEffect } from 'react';

interface UseFetchResult<T> {
  data: T | null;
  loading: boolean;
  error: Error | null;
}

function useFetch<T>(url: string): UseFetchResult<T> {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<Error | null>(null);

  useEffect(() => {
    fetch(url)
      .then(response => response.json())
      .then(setData)
      .catch(setError)
      .finally(() => setLoading(false));
  }, [url]);

  return { data, loading, error };
}
```
